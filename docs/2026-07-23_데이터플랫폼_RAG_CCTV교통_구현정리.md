## 1. 개요

기존 GeoAerospace(위성 궤도·항공기 시각화 + 자연어 지도제어)에 **지리·환경·교통 관측 데이터를 검색·근거화(RAG)하고, 실시간 CCTV 영상을 VLM으로 판독**하는 데이터 플랫폼을 추가했다. 핵심 설계 원칙은 일관되게 유지된다:

> **결정론 데이터 = 숫자(사실), VLM = 픽셀(시각 판단), LLM = 조율. 모델이 수치를 창작하지 않도록 검색된 사실로 그라운딩한다.**

이 원칙이 CCTV 교통 판독에서 특히 중요하게 작동한다 — 방면·방향·속도는 **표준노드링크 + ITS 실측(결정론)** 이 확정하고, VLM은 영상에서 보이는 혼잡을 **확인만** 한다.

## 3. 아키텍처 — RAG 3-레인 검색

관측/개념 데이터를 **서로 다른 3개 검색 레인**으로 다룬다. "래스터 픽셀은 임베딩 불가 → 텍스트만 임베딩" 이 핵심 통찰.

```
① 의미검색 (pgvector/bge-m3)  ← 개념 doc + 지역 요약카드(텍스트)
② 공간검색 (PostGIS)          ← FIRMS/OpenAQ/InSAR 포인트, moct_link 링크
③ 카탈로그 (STAC)             ← S2/SAR 장면 메타데이터
```

### 3.1 DB 인프라 (커밋 `31095e1`)
- `db/Dockerfile`: `postgis/postgis:16-3.4` + `apt postgresql-16-pgvector` → PostGIS·pgvector 단일 이미지
- `docker-compose.yml`: 호스트 포트 **5433**(로컬 5432 충돌 회피), 볼륨 영속, healthcheck
- `db/init/01-init.sql`: extensions + `knowledge_chunks(embedding vector(1024))` + `observations(geom Point 4326, GIST idx, dedup unique)`
- `lib/server/db.ts`: Pool 싱글턴 + `dbReady()` 프로브(미가용 시 인메모리 폴백) + `toVector()`

### 3.2 레인① — pgvector 의미검색 + 검색-증강 VLM (커밋 `d4e1b87`)
- `lib/knowledge.ts`: 14→23 청크(데이터소스 개념: InSAR·SAR·S2 Cloudless·S5P·LST·GLO-30·VWorld·STAC·OpenAQ)
- `lib/server/retrieve.ts`: **하이브리드 검색**(pgvector `<=>` 코사인 + 0.45×어휘부스트 재랭크), DB 없으면 인메모리 폴백
- `/api/reindex`: KNOWLEDGE → bge-m3 → `knowledge_chunks` upsert
- **retrieve-then-VLM**: `/api/analyze-image`가 질문+레이어로 개념 doc를 먼저 검색해 VLM 프롬프트에 "참고 개념(정의를 따를 것)"으로 주입 → VLM이 물리량 의미(NO₂ 컬럼≠지상농도, LST≠기온)를 오해하지 않게 함

### 3.3 레인② — PostGIS 공간검색 + 관측 인제스트 (커밋 `ff067f3`)
- `lib/server/spatial.ts`: `queryObservations(bbox)` = `geom && ST_MakeEnvelope(...)`(GIST 가속) + `summarizeObservations()`
- `lib/server/ingest.ts`: `ingestFires`·`ingestOpenAQ`·`ingestInsar` (dedup)
- 인제스트 라우트: `/api/ingest/{fires,openaq,insar,insar-ngu}`
- **analyze-image 공간 그라운딩**: bbox 관측을 자동 조회해 "참고 관측(공간DB 사실)"으로 VLM에 주입 → 화재·대기질 수치를 VLM이 지어내지 않고 인용

**FIRMS 오탐 감소** (커밋 `ff067f3` 내):
- `lib/server/demSample.ts`: AWS Terrarium DEM(z9) 타일을 sharp로 디코딩 → 고도 샘플
- `ingestFires`: 신뢰도 게이트(conf≥30, VIIRS 'low' 컷) + **바다 오탐 제거**(화재점 고도 ≤ −15m면 물로 판정). 검증: 서울 bbox 0건 / 동해안 bbox 19건(위치 필터 정확)

**실측 InSAR** (`fetchInsarNorway.ts`):
- InSAR Norway(NGU) 무인증 API: POST query → poll state → CSV 다운로드 → coherence 필터 + bbox + 다운샘플. 검증: Bergen 400점(최대 침하 2.9 mm/yr)

### 3.4 레인③ — STAC 장면검색 + 지역 요약카드 + describe_region (커밋 `a86c8d6`)
- `/api/stac`: Element84 earth-search v1 POST 검색(S2/SAR/Landsat, 구름% 정렬)
- **지역 요약카드(①↔② 브리지)**: `regionCard.ts`가 관측(레인②)을 텍스트 카드로 만들어 pgvector에 임베딩(레인①) → 의미검색이 "서울 대기질"에 실측 관측을 회수. `retrieve()`가 `knowledge_chunks UNION region_cards`로 블렌딩
- **describe_region 온디맨드**: `/api/region/describe` = 지오코딩→bbox→인제스트(best-effort)→카드생성·임베딩→개념검색→Qwen3 종합. 카드 TTL(기본 30분, 커밋 이후) 재사용
- 에이전트 배선(`lib/agent.ts` resolveIntent): `search_scenes`·`describe_region` 도구를 결정론 그라운딩으로 실행

## 4. 실시간 도로 CCTV

### 4.1 CCTV 레이어 (커밋 `6dec380`→`f192a0a`→`18b3866`)
- **데이터**: ITS `openapi.its.go.kr:9443/cctvInfo` — `coordx`=경도, `coordy`=위도(WGS84), `cctvurl`=HLS, `cctvname`
- ⚠️ 함정: 기존 data.go.kr 키는 ITS에서 거부(`4005`). data.go.kr CCTV는 실제로 ITS로 연계됨 → **데모키 `apiKey=test`가 무등록으로 실 CCTV 반환**(서울권 20 고정) / 실키는 전국 11,029개
- ⚠️ 함정: 실키=JSON, 데모키=XML 반환 → **양쪽 파싱**
- `lib/server/fetchCctv.ts` + `/api/cctv`(SSRF 프록시, 키 은닉) + `lib/cctvClient.ts`(지연 로딩)
- **렌더**: 처음 deck.gl ScatterplotLayer → 줌 중 깜빡임 → **maplibre 네이티브 circle 레이어**로 전환(GIBS 오버레이 패턴). 11k 마커는 **네이티브 클러스터**(`cluster:true`, clusterMaxZoom 11) — 클러스터 원+개수 라벨+개별 포인트 3레이어, 클러스터 클릭→`getClusterExpansionZoom` 줌인
- **팝업**: 우상단 기본위치(GeoAgent 버튼과 비겹침) + 헤더 드래그 이동

### 4.2 HLS 라이브 영상 (커밋 `11fed6e`)
- ITS 스트림(`cctvsec.ktict.co.kr`) 검사: 302→playlist.m3u8(토큰 2h), **전 구간 `Access-Control-Allow-Origin: *`** → **프록시 없이 hls.js 직접 재생**
- `components/CctvPlayer.tsx`: 동적 import(코드분할)·Safari 네이티브 HLS 폴백·LIVE 배지·자동재생 차단 시 ▶ 탭-투-플레이 폴백
- ⚠️ dev HTTP는 직접 재생 OK, 배포 HTTPS면 HTTP 스트림 mixed-content 차단 → 서버 프록시 필요(미구현)

## 5. CCTV 차량 혼잡도 VLM 판독 — 반복 정확화

가장 많은 반복 개선이 일어난 부분. 소형 VLM(7B)의 한계를 결정론 데이터로 보완하는 과정.

### 5.1 프레임 캡처 → VLM (커밋 `01a936b`)
- CctvPlayer에서 현재 프레임을 캔버스 캡처(hls.js는 MSE blob 소스라 canvas 비오염, `crossOrigin=anonymous`) → `/api/analyze-frame`(qwen2.5vl) → '원활/서행/정체' 정성 판단
- 설계원칙: **VLM은 시각 판단만, 정확한 차량 수는 VDS/실측이 담당**

### 5.2 방면별 구분 + 곡선·단방향 (커밋 `922bcf8`·`568d643`·`511d499`)
사용자 실사용 피드백으로 반복 수정:
1. **방면별 구분**: 왕복도로 좌/우 방면 따로 판정
2. **곡선 구간**: 중앙분리대가 비스듬 → 화면 반분할 금지, 실제 분리대 경계
3. **단방향 카메라**(IC진출/램프/터널): 없는 반대 방향 지어내던 것 → 카메라명·차도수로 단/양방향 판별
4. **앞/뒤 단서**: 차량 앞면=접근, 뒷면=멀어짐 으로 방향 확정

### 5.3 오판 원인 규명·수정 (커밋 `9e8ddc5`)
- **증상**: 둘 다 원활인데 "차량이 중앙분리대에 가까워서 정체"라는 **말 안 되는 근거**로 오판(실측 80km/h 무시)
- **원인**: 프롬프트가 중앙분리대를 과강조 → 7B VLM이 salient 개념(중앙분리대)에 꽂혀 **분리대 거리를 혼잡도 근거로 스퓨리어스 오용** + 방면별 억지 차등
- **해결**: ①중앙분리대는 방향 경계일 뿐 혼잡도와 무관 명시 ②혼잡도는 차량 밀도·흐름으로만 ③**실측 속도를 기본상태로**(speedLevel: 고속도로 80/40, 일반 30/15 기준) 삼고 명백한 정체만 예외 ④애매하면 양쪽 동일 등급
- **교훈**: 소형 VLM은 과지시하면 salient 개념에 꽂혀 오추론 → **기준을 구체·단순화하고 결정론 데이터를 앵커**로

### 5.4 실측 속도 실증(RAG) (커밋 `511d499`·`8530d7b`)
- `lib/server/fetchTraffic.ts`: ITS `trafficInfo`(실시간 소통, `roadName`·`speed`·`linkId`)를 CCTV 근처 조회. ⚠️ 실키=JSON/데모키=XML 양파싱
- 도로별 그룹 평균 → CCTV명 힌트("경부선"→경부고속도로) 매칭 대표 도로 실측속도를 VLM 프롬프트에 주입 + UI 표시

## 6. 표준노드링크 — 방면별 정밀 속도 매칭 (커밋 `256adfc`)

**문제 인식**: 소형 VLM이 방면 지명(세종→서울)을 **오독**. → 방면은 데이터로 확정하고 VLM은 확인만 하게.

### 적재
- 사용자 제공 `MOCT_LINK.shp`(263MB, EPSG:5186 ITRF2000 Central Belt)
- ⚠️ 함정: **docker cp가 Windows에서 "closed pipe" 반복 실패** → **Node `shapefile` 라이브러리로 호스트에서 직접 pg 적재**(`db/load_nodelink.mjs`, euc-kr 인코딩, ROAD_RANK 101~104 고속·국도·광역시도만 **427,013개** 필터)
- `moct_link(link_id, f_node, t_node, road_name, geom LineString 5186)` + GiST 인덱스
- ⚠️ 컨테이너에 `shp2pgsql` 없음 → `apt install postgis`로 설치 가능하나, 결국 Node 경로 채택

### 매칭 파이프라인 (`fetchTraffic.ts`)
```
CCTV lon/lat → ST_Transform(4326→5186) → 최근접 link → 같은 road_name 근처 링크(ST_DWithin 1km)
  → ST_Azimuth 방위로 두 방향 분리(base±90°) → 각 방향 linkId를 trafficInfo 속도맵과 매칭
  → 방향별 평균속도 + 8방위 라벨(남행/북행/…)
```
- `LINK_ID` = ITS `trafficInfo.linkId` 일치 확인됨
- 프롬프트: 방위별 실측을 기준선으로 주고, 화면 방면 지명을 방위(북행=북쪽…)에 대응시키게

### 실습(검증)
- **남사정류장(경부고속도로)** → 최근접 링크 2개: bearing 340°(북행) 11m / 160°(남행) 31m
- linkId 매칭: **남행 78 km/h = 부산 방면**, **북행 95 km/h = 서울 방면** (경부 남=부산, 북=서울 정확)
- VLM: "부산 방면: 원활 / 서울 방면: 원활" — **화면 지명을 데이터 방위에 정확 대응, 세종→서울 류 오독 해소**
- UI: `📡 실측 경부고속도로 남행 78·북행 95 · ITS+노드링크`

## 8. 함정·교훈 모음

- **소형 VLM 프롬프트**: 과지시하면 salient 개념(중앙분리대)에 꽂혀 스퓨리어스 추론 → 기준 구체·단순화 + 결정론 데이터 앵커
- **VLM은 방면 라벨 오독** → 방면·방향은 표준노드링크로 확정, VLM은 시각 확인만(설계원칙 준수)
- **ITS 키**: data.go.kr 키 ≠ ITS 키. 데모키 `test`가 실데이터(제한). 실키=JSON/데모키=XML 양파싱
- **HLS**: CORS `*`라 프록시 불필요(dev). 배포 HTTPS는 mixed-content 프록시 필요
- **docker cp** Windows에서 대용량/일부 경로 "closed pipe" 실패 → Node로 직접 적재 우회
- **deck.gl vs 네이티브**: globe/terrain 줌 중 deck 레이어 깜빡임 → maplibre 네이티브 레이어가 안정
- **maplibre 소스 포맷**: 실키 JSON / 데모키 XML, getType 무시되는 경우 있음 → 매직바이트·시작문자로 분기
- **dev 서버 불안정**: 세션 중 반복 크래시(HTTP 000) — HMR 스테일도 잦음. 실사용 검증은 재시작 후
- **자동화 한계**: 스크린샷은 globe GPU로 타임아웃, VLM 추론 30s>툴한계 → window 전역 폴링으로 우회
