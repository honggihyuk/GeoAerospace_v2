# 고도화 구상 — 궤도 오차 최소화 + 실시간 3D 실제 모습

> 현재 구현(SGP4 + 라이브 TLE + 3D 우주 뷰)을 전제로, 두 목표의 고도화 방법을 구상한다.
> **핵심 전제:** 궤도 오차는 "0"이 될 수 없다(모든 예측에 오차 존재). 정석은 **최소화 + 정량·시각화**.

---

## A. 위성 궤도 오차 최소화 + 정량화

### 오차의 원천 (현재)
| 원천 | 크기 | 현 상태 |
|---|---|---|
| TLE 노후화 | ~1–3 km/일 누적 | SatNOGS 폴백(CelesTrak 차단 환경), 4h 캐시 |
| SGP4 모델 한계 | epoch에서 ~1 km, 시간당 증가 | satellite.js SGP4 (골든벡터 검증됨) |
| 프레임/시각 근사 | 수십 m~수백 m | GMST 사용(GAST·세차·장동·극운동 미적용) |
| 기동(maneuver) | km급 점프 | 미감지 |

### A1. 더 신선하고 권위 있는 원소 — *즉효, 낮은 노력*
- **Space-Track.org**(18 SDS 권위 카탈로그) 계정 도입 → 가장 신선한 원소. CelesTrak GP 1차 복구(현 차단은 테스트망 이슈 → 사용자망 재확인/프록시), SatNOGS 폴백.
- **최신 epoch 선택**: 위성당 여러 TLE 중 epoch가 현재에 가장 가까운 것 사용.
- **갱신 주기 단축**: 4h → 활성 LEO(드래그 큰)에는 1–2h. epoch가 오래되면 자동 재수집.
- **OMM(CCSDS)** 포맷 채택(같은 SGP4 원소, 신형).

### A2. 고정밀 전파 (SGP4 넘어서기) — *중간 노력, 큰 향상*
- **서버 수치 전파 마이크로서비스**(Orekit / hapsira, Python): EGM 고차 중력장 · 대기항력(우주기상 F10.7/Ap) · 태양복사압 · 3체(달·태양) 섭동 → **시간당 sub-100 m**. TLE→osculating 상태 변환 후 수치적분.
- **정밀 프레임 변환**: GMST → 완전 **IAU 세차·장동 + GAST + 극운동(IERS EOP)**. Skyfield/Astropy 서버로 프레임 오차 제거(수십 m).
- **2계층 전파**: 클라 SGP4(실시간 렌더, 빠름) + 서버 수치전파(정밀 통과예측·해석, 느림·정확).

### A3. 실측 궤도 · 관측 융합 — *높은 노력, 최고 정확도*
- **공개 정밀 ephemeris 우선 사용**: ISS는 **NASA OEM 상태벡터**(TLE보다 훨씬 정확), GNSS는 **IGS SP3**. 있으면 TLE 대신 사용.
- **관측 융합 궤도결정(OD)**: SatNOGS Doppler·광학/레이더 관측 → 최소자승/칼만필터로 궤도 정밀화(진짜 SDA). 후속.
- **기동 감지**: TLE 급변 탐지 → 즉시 재수집 + 사용자 경보.

### A4. 불확실성 정량·시각화 — *즉효, 신뢰성 ↑*
- **TLE 나이 배지**(이미 있음: LIVE·source) + **오차 성장 모델**(나이×성장률) → 위성에 **불확실성 타원체(covariance)** 렌더.
- **다중 소스 교차검증**: CelesTrak vs Space-Track vs SatNOGS 전파 결과 스프레드 = 실측 불확실성 지표.
- **정확도 CI 확장**: SGP4 vs 수치전파 vs 공개 ephemeris 상시 비교(기존 골든벡터 테스트 확장).

## B. 실시간 3D "실제 모습" 고도화

### B1. 포토리얼 실시간 지구 (3D 우주 뷰)
- 3D 지구 구에 **NASA GIBS 당일 트루컬러**(VIIRS/MODIS, 매일 갱신) 텍스처 → "오늘의 지구".
- **야간 도시광** 텍스처(밤면) + **실시간 주야 터미네이터**(태양 ECI 위치 계산) + **구름 레이어**(GIBS NRT, 반투명 오버레이 구).
- 대기 산란 셰이더 업그레이드(현 프레넬 → Rayleigh/Mie 근사, 골든아워 림).

### B2. 진짜 실시간 + 시간 제어
- **타임 컨트롤러**(목업의 타임 스크러버 구현): 실시간(1×)/일시정지/배속(60×·3600×)/스크럽(과거·예측 ±시간). 궤도 진화·통과 예측을 직접 관찰.
- 60fps 부드러운 보간, **"현재 위치" 강조**(펄스 마커·라이브 지상궤적 연장).

### B3. 실제 위성 3D 모델 (glTF)
- 명명 위성에 실제 glTF(ISS·Hubble·범용 버스), **자세 정렬**(nadir 지향 + 태양전지판 태양 추적). 현 3D 뷰의 구(sphere) → 실모델 승격.

### B4. 실제 상태 오버레이 (실시간)
- **일조/식(eclipse)**: 지구 그림자 판정 → 위성이 햇빛/그림자 중 어디인지 색·밝기로 표시.
- **지상국 가시성 · 커버리지 풋프린트**: 위성이 현재 보는 지표 원(footprint) + 가시 지상국.
- **라이브 통과 카운트다운**(선택 지점 기준).
- **라이브 텔레메트리**: SatNOGS Network 관측 데이터(일부 위성 실제 다운링크 디코딩).

### B5. 실시간 데이터 파이프라인
- **WebSocket/SSE 푸시**(설계서 §7.2 DataBus): 서버가 권위 상태를 전파·푸시 → 다중 클라 동기화 + 계산 오프로드. 또는 현 클라 전파 유지 + 주기 TLE 갱신.

## D. 지금 바로 구현 권장 (highest ROI, 현 코드 확장)

1. **정확도 즉효(A1+A4)**: 위성당 최신 epoch TLE 선택 + 갱신 주기 단축 + **TLE 나이/추정오차 배지**(HUD·위성 툴팁). 오래된 TLE는 회색·경고.
2. **실시간 실제감(B1+B4)**: 3D 우주 뷰에 **실시간 주야 터미네이터 + 태양 방향 조명 + eclipse(식) 판정**(위성이 그림자면 어둡게). 태양 위치는 클라에서 계산.
3. **타임 컨트롤러(B2)**: 재생/일시정지/배속(1×·60×·3600×)로 궤도·통과를 가속 관찰. `now`를 가상 시계로 대체.

→ 이 3개는 기존 satellite.js/Three.js/스토어 위에 얹어 **며칠 단위로 구현 가능**하며, "오차를 정직하게 줄이고 실시간 실제 모습을 보여준다"는 목표를 즉시 체감시킨다. A2(서버 수치전파)·B1(GIBS 텍스처)는 그다음 단계.
