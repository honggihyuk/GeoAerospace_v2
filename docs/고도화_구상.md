# 고도화 구상 — 궤도 오차 최소화 + 실시간 3D 실제 모습

> 현재 구현(SGP4 + 라이브 TLE + 3D 우주 뷰)을 전제로, 두 목표의 고도화 방법을 구상한다.
> **핵심 전제:** 궤도 오차는 "0"이 될 수 없다(모든 예측에 오차 존재). 정석은 **최소화 + 정량·시각화**.

> **갱신 이력**
> - 2026-07-20: A1·A2·A3·B1·B3·B4 구현 완료. **원안의 두 전제가 실측으로 반증**되어 §0에 근거와 함께 기록하고, 해당 절(A2·A3)을 정정했다.

## 0. 실측으로 확정된 사실

*이 절은 구상이 아니라 **측정 결과**다. 아래 숫자는 추정이 아니라 실제 관측이며, 이후 판단의 근거가 된다.*

### 0.1 SGP4의 실제 오차 (기준: NASA ISS OEM, CCSDS/EME2000, NASA/JSC 생성)

TLE epoch 2026-07-19T19:56 기준, 시간에 따른 위치 오차 (단위 km):

| Δt | 총오차 | radial | **along-track** | cross |
|---|---|---|---|---|
| 0 h | **0.681** | −0.045 | **0.649** | −0.201 |
| 24 h | 0.623 | −0.033 | 0.622 | 0.023 |
| 72 h | 0.718 | 0.058 | 0.621 | −0.357 |
| 120 h | 3.090 | 0.072 | **−3.007** | −0.710 |
| 168 h | 9.877 | 0.109 | **−9.803** | −1.200 |
| 240 h | 28.504 | 0.019 | **−28.441** | 1.888 |

- **epoch 시점에 이미 681 m 어긋나 있다.** 이 값은 TLE 자체에 들어 있어 전파 방식으로 제거할 수 없다.
- 오차는 거의 전부 **along-track(궤도 위상)** 이다. 프레임 무관 검증(\|r\| 비교)에서 반경 차이는 1.9~124 m에 불과했다 → 고도는 잘 맞고 **궤도상 위치**가 어긋난다.
- **증가율은 단일 수치로 말할 수 없다.** 3일까지 거의 평평하다가 이후 급증하는 비선형 곡선이라, 선형 적합값이 구간에 따라 100배 넘게 달라진다:

  | 적합 구간 | 증가율 |
  |---|---|
  | 0~72 h | 0.02 km/일 |
  | 0~168 h | 1.20 km/일 |
  | 0~240 h | 2.58 km/일 |

  → 앱이 표시하는 값(0~168h 기준, 1.2~1.5 km/일)과 원안 표의 "1–3 km/일" 추정은 모두 이 범위 안이다. **인용할 때는 반드시 구간을 함께 밝힐 것.**

> *측정 방법 주의:* OEM 표본은 4분 간격이라 **보간 방식이 결과를 오염시킨다.** 2점 Hermite는 이 간격에서 고유 오차가 `R·N⁴h⁴/384 ≈ 95 m`로, 수백 m를 재면서 95 m 오차를 안고 가게 된다. 위 값은 **8점 Lagrange**(내부 구간 ~20 cm)로 재측정한 것이다.

### 0.2 프레임 차이의 크기

- **TEME(SGP4 출력) ↔ J2000(EME2000)** 회전: **0.19~0.37°** = 6800 km에서 **23~44 km**. J2000 이후 26년치 세차 때문이며, 두 프레임 좌표를 변환 없이 비교하면 결과가 전부 무의미해진다.
- 구현한 IAU-76 세차 + IAU-80 장동(상위 20항)은 Skyfield(전항 구현) 대비 **잔차 0.8 m**.
- *미검증:* 극운동(PEF→ITRF)은 여전히 미적용. 지표 좌표에 남는 오차 크기는 측정하지 않았다.

### 0.3 데이터 소스 실태 (이 네트워크 기준)

| 소스 | 상태 | 비고 |
|---|---|---|
| CelesTrak | **차단** (무응답, 21 s 타임아웃) | 1차 소스로 쓸 수 없음 |
| Space-Track | 정상 | 전 카탈로그 커버. 분당 30 / 시간당 300 요청 제한 |
| SatNOGS DB | 정상 | **아마추어 무선 위성만** — Starlink·KOMPSAT 등은 아예 없음(`[]` 반환) |
| SatNOGS Network | 정상 | 지상국 4,370곳(온라인 314), 지상국별 `min_horizon` 제공 |
| NASA ISS OEM | 정상 | AWS S3 공개. 15일치·4분 간격·718 KB |
| NASA GIBS | 정상 | WMS로 전지구 등장방형 텍스처 1회 요청 가능 |

### 0.4 그 밖에 실측으로 드러난 것

- **GIBS 당일 트루컬러에는 결손 경도대가 있다.** 2026-07-19 기준 −180°~−133.8°(46°폭)가 통째로 비어 있었다. "오늘" 요청 시 대부분 미수집 상태로 오는 것과는 별개 문제다.
- **소멸한 위성을 추적하고 있었다.** 데모 세트의 STARLINK-1007(44713)은 **2024-10-02 재진입**(SATCAT `DECAY`). 소멸 객체는 권위 카탈로그도 마지막 원소만 주므로 어떤 소스를 붙여도 영구히 655일+ 낡은 상태였다. → STARLINK-3120(49419)으로 교체.
- **ISS 재부스트는 TLE 이력에서 명확히 보인다.** 70일간 2건(2026-06-10 +1.55 km, 2026-07-01 +1.83 km). 항력 변화(+0.00015 rev/day/day) 대비 **20~40배 크기에 부호가 반대**라 분리가 쉽다.

## A. 위성 궤도 오차 최소화 + 정량화

### 오차의 원천 (2026-07-20 갱신)
| 원천 | 크기 | 상태 |
|---|---|---|
| TLE 노후화 | 3일까지 sub-km, 이후 급증 (§0.1) | ✅ Space-Track 1차 + 90분 갱신 |
| SGP4 모델 한계 | **epoch에서 681 m**, along-track 지배 (실측) | ✅ ISS는 정밀 ephemeris로 대체 |
| 프레임/시각 근사 | TEME↔J2000 **23~44 km** (실측) | ✅ IAU-76/80 변환 구현 (잔차 0.8 m) |
| 기동(maneuver) | ISS 재부스트 **1.5~1.8 km** (실측) | ✅ TLE 이력에서 감지 |

### A1. 더 신선하고 권위 있는 원소 — *즉효, 낮은 노력* — ✅ **구현 완료**
- **Space-Track.org** 도입 완료. 세션 쿠키 90분 재사용 + 전 위성 1회 배치 쿼리로 레이트리밋(분당 30 / 시간당 300) 보호.
- 갱신 주기 4h → **90분**. 뷰 전환과 무관하게 동작(이전에는 2D 컴포넌트에 묶여 있어 3D로 전환하면 영구히 로딩 상태로 남는 버그가 있었다).
- CelesTrak은 **복구 실패**(이 네트워크에서 차단). SatNOGS는 커버리지 한계로 폴백 전용.
- 소스 표기를 정직하게: 일부라도 데모/스테일이 섞이면 `LIVE`가 아니라 `PARTIAL`로 표시.
- *미구현:* OMM(CCSDS) 포맷 채택 — 현재 3LE로 충분해 우선순위를 낮췄다.

### A2. 고정밀 전파 — ⚠️ **원안의 핵심 전제가 반증됨**

> **원안:** 서버 수치 전파 마이크로서비스(Orekit/hapsira)로 TLE→osculating 변환 후 수치적분 → **시간당 sub-100 m**.
>
> **실측 결과: 불가능하다.** (근거 §0.1)
> 1. 수치적분은 **초기 상태보다 정확해질 수 없다.** SGP4는 TLE epoch 시점에 이미 681 m 어긋나 있고, 그 오차는 TLE에 들어 있어 어떤 적분기로도 제거되지 않는다. sub-100 m는 이 하한 아래다.
> 2. TLE 평균원소는 **SGP4 이론에 종속적**이다. osculating으로 변환해 다른 힘모델로 적분하면 오차가 줄기는커녕 이론 간 불일치가 더해진다.
> 3. 오차가 거의 전부 along-track이라는 점도 이를 뒷받침한다 — 고차 중력장·항력 모델링으로 해결될 성질이 아니다.
>
> → **Python 마이크로서비스를 만들지 않았다.** 거짓 정밀도를 생산할 뿐이다.
> → sub-100 m를 원하면 정밀 ephemeris를 직접 쓰는 수밖에 없다(§A3).

- **정밀 프레임 변환** — ✅ **구현 완료**. IAU-76 세차 + IAU-80 장동(상위 20항) + 분점방정식으로 TEME↔J2000. Skyfield 대비 잔차 0.8 m. *원안의 "수십 m 개선"이 아니라 **23~44 km** 규모였다* — 정밀 ephemeris를 쓰려면 필수 전제였다.
- **2계층 전파** — ✅ 취지대로 구현. 클라 SGP4(실시간 렌더) + 서버 정밀 소스(정확도 산출·대조). 단 서버 쪽은 수치전파가 아니라 **실측 ephemeris**다.

### A3. 실측 궤도 · 관측 융합 — *높은 노력, 최고 정확도*

- **공개 정밀 ephemeris 우선 사용** — ✅ **구현 완료**. NASA ISS OEM을 ±8시간 창(22 KB, 240표본)으로 잘라 클라에 전달, 8점 Lagrange 보간 + J2000→TEME 변환 후 **렌더 좌표로 직접 사용**. 커버 밖이면 SGP4로 자동 폴백.
  - 검증: 씬 좌표 ↔ 정밀 ephemeris **0 km**(완전 일치), ↔ SGP4 0.377 km. 서버 측 독립 측정 0.380 km와 일치.
  - *미구현:* GNSS용 IGS SP3.
- **관측 융합 궤도결정(OD)** — ❌ **현재 데이터로는 불가능** (원안도 "후속"으로 표시)
  > SatNOGS 관측 레코드를 실제로 확인한 결과 **궤도결정에 쓸 측정값이 없다.**
  > - `rise_azimuth`·`max_altitude`·`set_azimuth`는 관측 *예약*을 위해 **TLE로 계산한 예측값**이다. 이를 OD에 넣으면 TLE로 만든 값을 TLE에 되먹이는 순환이 된다.
  > - 측정 산출물은 워터폴 **이미지**와 복조 프레임뿐이다. 보정된 주파수–시간 계열이 없어 Doppler 잔차를 얻으려면 이미지에서 곡선을 추출하는 별도 연구가 필요하다.
  >
  > → 진행하려면 **다른 데이터원**(광학 추적, 레이더, 또는 자체 수신 설비)이 전제되어야 한다.
- **기동 감지** — ✅ **구현 완료**. Space-Track `gp_history` 70일치에서 반장축 계단 상승을 탐지. 임계값을 고정 상수가 아니라 **관측된 항력 잡음(MAD)에서 유도** — 위성마다 고도·면적비가 달라 ISS와 정지궤도에 같은 상수를 쓸 수 없다. ISS 재부스트 2건을 σ 308·38로 검출.
  - *미구현:* 감지 시 자동 재수집 트리거 및 사용자 경보(현재는 표시만).

### A4. 불확실성 정량·시각화 — ✅ **구현 완료 (원안보다 강하게)**
- 원안은 "오차 성장 **모델**(나이×성장률)"이었으나, ISS는 **실측값**으로 대체했다 — NASA OEM 대조로 `along 933m · radial 45m · 증가율 1.37 km/일`처럼 실제 차이를 표시. 정밀 ephemeris가 없는 위성만 추정치(`~±Nkm`)로 폴백.
- *미구현:* 불확실성 타원체(covariance) 3D 렌더, 다중 소스 교차검증 스프레드.

## B. 실시간 3D "실제 모습" 고도화

### B1. 포토리얼 실시간 지구 — ✅ **구현 완료**
- NASA GIBS 당일 트루컬러 + 야간 도시광(Black Marble) + 태양각 블렌딩 셰이더.
- **결손 대응이 핵심이었다**(§0.4). 당일 영상은 스와스 갭이 있어 그대로 쓰면 검은 쐐기가 생기고, 알베도 0인 영역에 박명 띠가 더해져 주황 얼룩까지 남는다. → **당일 → 전날 → 정적 BlueMarble** 3단 폴백으로 해결. 정적 베이스만으로는 구름 유무 차이로 이음매가 드러나 부족했다.
- 대기 산란: 프레넬 → Rayleigh/Mie 근사 + 터미네이터 골든아워 링.
- *미구현:* 별도 구름 레이어(트루컬러에 이미 구름이 포함돼 중복).

### B2. 진짜 실시간 + 시간 제어 — ✅ **기구현** (재생/일시정지/배속/스크럽)

### B3. 실제 위성 3D 모델 — ✅ **구현 완료 (단, glTF 아님)**
- **스케일 현실:** 씬은 1 unit = 1000 km이고 ISS 실제 크기는 109 m = 0.000109 unit이다. **실축척 glTF는 서브픽셀이라 보이지 않는다.** 기존 구 마커도 이미 1,100배 과장돼 있었다. 이 배율에서 위성은 화면상 10~30픽셀이라 고밀도 메시의 디테일은 대부분 버려진다.
- → **절차적 모델**(ISS 트러스+모듈+어레이 / Hubble / 범용 버스) + GLTFLoader 연결부. 실제 glTF는 `public/`에 넣어 교체 가능.
- **자세 정렬**: LVLH 프레임(`z = -r̂` 천저, `y = -ĥ`, `x = y×z` 비행방향) + 태양전지판 Y축 태양 추적.
- **마커 LOD 필요**: 모델만 두면 넓은 뷰에서 식(eclipse) 중인 10픽셀 모델이 어두운 지구에 묻혀 추적 기능을 잃는다. 화면 크기 고정 글로우 마커를 병행한다.

### B4. 실제 상태 오버레이 — ✅ **대부분 구현**
- 일조/식(eclipse) — 기구현. *주의: three의 DirectionalLight는 지구에 의한 가림을 모르므로 식 판정을 재질에 직접 반영해야 한다.*
- **커버리지 풋프린트** — ✅ `λ = acos((Re/(Re+h))·cos ε) − ε`. ISS 428 km에서 20.44°(지표 반경 2273 km).
- **지상국 가시성** — ✅ SatNOGS Network 온라인 314곳. **지상국별 `min_horizon`을 존중**해야 한다 — 실측 중앙값 10°에 범위 0~70°라, 일괄 0°로 판정하면 지평선이 70°인 곳도 "보인다"고 표시된다.
- **라이브 통과 카운트다운** — ✅ AOS/peak/LOS 탐색(조밀 탐색 + 이분법).
- **라이브 텔레메트리** — ❌ **미구현**. SatNOGS 관측 엔드포인트는 도달 확인했으나 관측 목록·복조 데이터 표시는 별도 UI 설계가 필요.

### B5. 실시간 데이터 파이프라인 — ❌ **미구현**
- 현재는 클라 전파 + 주기 갱신 방식을 유지 중(원안의 대안 경로). WebSocket/SSE 푸시는 다중 클라 동기화가 필요해질 때 검토.

## D. 남은 작업

우선순위 순:

1. **B4 라이브 텔레메트리** — SatNOGS 관측 목록·복조 프레임 표시. 유일하게 남은 원안 항목.
2. **기동 감지 후속** — 감지 시 TLE 자동 재수집 트리거 + 사용자 경보(현재는 표시만).
3. **A4 잔여** — 불확실성 타원체 3D 렌더, 다중 소스 교차검증 스프레드.
4. **극운동(polar motion)** — 지표 좌표에 남은 오차. 먼저 **크기를 측정**하고 값어치가 있는지 판단할 것(§0.2).
5. **IGS SP3** — GNSS 위성용 정밀 ephemeris. ISS OEM과 같은 경로를 재사용할 수 있다.

**착수 전 확인:** §0의 실측값은 2026-07-20 기준이다. 데이터 소스 상태(§0.3)는 네트워크 환경에 따라 달라질 수 있으니, CelesTrak 차단 등은 재확인 후 판단할 것.
