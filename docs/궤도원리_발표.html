궤도를 "추정"한다는 것 = SGP4 전파

두 뷰는 똑같은 계산으로 위성 위치를 구한다. 통계적 추측이 아니라, 물리 모델로 지금 위성이 어디 있는가를 전파(propagation)하는 것이다.

DATA · 입력

TLE (두 줄)

경사각·이심률·평균운동 등. 일반 케플러가 아닌 SGP4 전용으로 피팅된 평균 궤도요소.

→
ENGINE · 이론

SGP4 / SDP4

지구 편평률(J2·J3·J4)·대기 항력(B*)을 해석적으로 근사하는 섭동 이론. Hoots&Roehrich 1980 · Vallado 2006.

→
OUTPUT · 좌표

ECI (TEME) 위치·속도

결과는 관성계 좌표. twoline2satrec → propagate

▽ 여기서 두 뷰가 갈린다 — 차이는 단 하나, 지구 자전각 GMST

GMST를 위성에 적용

ECI → ECEF → 경위도(측지). 지구고정계로 내려 2D 지도에 올린다. → 지상궤적(사인파)

GMST를 지구에만 적용

ECI 좌표를 변환 없이 3D 씬에 그대로 배치. 위성은 안 돌리고 지구만 돌린다. → 닫힌 타원

ECI(관성계) ↔ ECEF(지구고정)의 유일한 차이 = GMST(그리니치 평균 항성시) 회전 · satellite.gstime(date)

LAYER A · 2D GLOBE

지구고정계 (ECEF)

  • 원리: 지구가 궤도 아래에서 자전 → 지상궤적이 서쪽으로 밀려 사인파형
  • 변환: eciToGeodetic(pos, gmst) → lat·lon·alt(WGS-84)
  • 산출: 궤도 링(고도 유지) + 지상궤적(height=0) · ±180° 분할

LAYER B · 3D 실축척

관성계 (ECI)

  • 원리: ECI는 관성계라 회전 없음 → 궤도는 고정된 닫힌 타원, 지구만 돈다
  • 배치: ECI 직접 사용 (x,z,−y)×.001 · earth.rotation.y=gmst
  • 스케일: 1 unit = 1000 km, R=6.371 → LEO가 지구에 밀착한 실축척
