# GeoAgent 질문 프롬프트 가이드

> 이 문서는 **실제 구현된 라우팅 규칙(`lib/agent.ts`)에 근거**합니다. 추측이 아니라 코드가 실제로 무엇을 트리거하는지 정리한 것입니다.

## 0. 먼저 알아둘 동작 원리

이 에이전트는 **LLM이 도구를 고르지 않습니다.** 정규식 기반 결정론 라우팅(`resolveIntent`)이 도구를 정하고, 계산은 전부 서버가 하며, **Qwen은 이미 확정된 수치를 문장으로 풀어쓰기만** 합니다.

```
질문 → [정규식 라우팅] → [결정론 계산] → [Qwen 서술] + [원본 수치 블록]
                                              ↑ 없어도 답은 나옴    ↑ 항상 그대로 표시
```

실무적으로 중요한 함의:

- **정해진 표현을 쓰면 확실히 동작하고, 벗어나면 라우팅이 안 됩니다.** 창의적으로 돌려 말하면 일반 RAG 질문으로 빠집니다.
- **수치는 항상 원본 블록이 함께 나옵니다.** 서술문과 수치가 다르면 **원본 블록이 정답**입니다.
- **품질 경고(⚠️)가 붙으면 서술문이 생략됩니다.** 신뢰할 수 없는 수치를 매끄러운 문장으로 포장하지 않기 위한 의도된 동작입니다.

## 1. 분광지수 — 식생 / 수체 / 연소

### 트리거 (셋 중 하나 + 장소)

| 지수 | 트리거 단어 | 예시 |
|---|---|---|
| **NDVI**(식생) | 식생, 녹지, 초목, 산림 면적/비율/분포, 숲 면적/비율/분포, `ndvi` | `서울 식생 알려줘` |
| **NDWI**(수분) | 수체, 수면, 물 면적/비율, 호수 면적, 저수량/저수율, `ndwi` | `소양호 수면 알려줘` |
| **NBR**(연소) | 연소 지수/정도, 화상, 소실 면적, 산불 피해/흔적/자국, `nbr` | `강릉 산불 피해 알려줘` |

**✅ 잘 되는 질문**
```
서울 식생 알려줘
Central Park 식생 상태 보여줘
부산 녹지 비율 알려줘
```

**❌ 안 되는 질문과 이유**
```
서울 나무 많아?          → '나무'는 트리거 아님 (식생/녹지/산림 중 하나를 쓸 것)
식생 알려줘              → 장소가 없음 (bbox를 만들 수 없음)
전국 식생 알려줘         → 'AOI가 너무 큼' (1°×1° 상한)
```

### 장소 지정 요령

- 장소는 **지오코딩으로 시스템이 좌표·경계를 만듭니다**(LLM이 좌표를 지어내지 못하게).
- **행정경계·공원처럼 폴리곤이 있는 곳은 실제 모양으로 잘라냅니다.**
  실측: `Central Park` → 사각형 9.09 km²(맨해튼 거리 포함) → **폴리곤 3.09 km²**(실제 3.41), 초고밀도 식생 55.4% → **90.2%**
- 후보가 여럿이면 **면적이 가장 큰 것**을 고릅니다(`울진` → 울진역 1 km²가 아니라 울진군 174 km²).

## 2. 두 시점 비교 — 가뭄·수위·증감

### 트리거: 지수 트리거 + **날짜 2개(YYYY-MM-DD)** + 비교 의도

```
폴섬호 수면 2021-09-15 2022-09-15 비교
```

> ⚠️ 현재 이 조합은 **API(`/api/spectral/compare`)로는 되지만 채팅 라우팅은 변화탐지로 갑니다.**
> 채팅에서 날짜 2개를 주면 `change_detection`이 우선 잡습니다. 지수 증감만 보려면 API를 직접 쓰세요.

### 수체 면적을 볼 때 (중요)

**NDWI 임계값은 물을 제대로 못 잡습니다.** L2A가 어두운 수면의 반사율을 음수로 과보정해 NDWI가 −1로 뒤집힙니다(실측: 폴섬호 수체 0.02 km², 실제 만수위 46 km²).
→ **수체 면적은 `수체(SCL 기준·권장)` 값을 보세요.** 실측 폴섬호 2021-09 **25.42 km²** → 2022-09 **36.64 km²** (+44.2%).

## 3. 변화 탐지 — 산불 피해·개발·벌목

### 트리거: 변화 의도 + 장소 + **날짜 2개 필수**

트리거 단어: `변화`, `전후`, `피해 면적/규모/정도`, `dnbr`, `연소 심각도`, `비교`

```
✅ 울진 변화 2022-03-03 2022-03-20
✅ Pacific Palisades 피해 규모 2025-01-05 2025-01-28
❌ 울진 변화 알려줘            → 날짜가 없음
```

**날짜가 필수인 이유**: 사건 전/후를 시스템이 임의로 정할 수 없습니다. 잘못 정하면 계절 변화를 사건으로 오독합니다(실측: 2월 vs 4월 비교에서 개엽이 연소 신호를 상쇄).

### 날짜를 고르는 요령 (결과 품질을 좌우)

1. **사건에 바짝 붙이세요.** 각 날짜 기준 과거 30일에서 **가장 가까운** 장면을 고릅니다.
   - 좋음: 화재 3/4~3/13 → `2022-03-03` / `2022-03-20`
   - 나쁨: `2022-02-28` / `2022-04-15` (계절 변화가 섞임)
2. **같은 계절로 맞추세요.** 연 단위 비교라면 같은 달을 쓰는 게 안전합니다(`2021-09-15` / `2022-09-15`).
3. **구름 적은 시기를 고르세요.** 한국 7~8월(장마)은 거의 실패합니다.

### 결과 읽는 법

| 지표 | 의미 |
|---|---|
| **합성 변화** | "무엇이든 바뀐 정도" (NDVI .35 / NDWI .15 / NBR .15 가중합) |
| **dNBR** | 연소 심각도. **양수 = 탐** (NBR 전 − 후) |
| **valid_fraction** | 유효 픽셀 비율. **30% 미만이면 경고**가 붙고 서술이 생략됩니다 |

**실증된 결과**
```
Dixie Fire (2021-07-08 → 09-16)       dNBR +0.615, 고심각도 46.8%
Pacific Palisades (2025-01-02 → 01-17) dNBR +0.599, 고심각도 40.2%
```

## 4. 그 밖의 도구

| 하고 싶은 것 | 이렇게 물어보세요 |
|---|---|
| 위성 추적 | `ISS를 추적해줘`, `천리안 보여줘`, NORAD 번호 |
| 지도 이동 | `파리로 이동`, `도쿄 상공을 보여줘` |
| 레이어 On/Off | `항공기 레이어를 꺼줘`, `3D 지형 켜줘` |
| 활성 화재(FIRMS) | `캘리포니아 산불 표시해줘`, `FRP 100MW 이상` |
| 장면 검색(STAC) | `서울 위성영상 찾아줘`, `구름 10% 이하 장면` |
| 지역 브리핑 | `서울 대기질 어때`, `부산 상황 브리핑` |
| 영상 해석(VLM) | `연기 확산 방향 분석해줘` (트루컬러/bands721 오버레이 필요) |
| 개념 질문 | `TLE가 뭐야?`, `SAR로 지반침하를 어떻게 관측하나?` → RAG |

> **산불 관련 주의**: `산불 보여줘`는 FIRMS 포인트, `산불 피해`는 NBR로 갈립니다. 의도에 맞는 단어를 쓰세요.

## 5. 자주 겪는 실패와 대처

| 증상 | 원인 | 대처 |
|---|---|---|
| `AOI가 너무 큼` | 지수 1°, 변화 0.5° 상한 | 더 좁은 지명으로 |
| `조건에 맞는 장면 없음` | 구름·기간 | 다른 계절, `cloud` 완화 |
| `유효 픽셀 N%` 경고 | 구름·연기·지형그늘 | 다른 날짜. **수치를 신뢰하지 말 것** |
| 수체가 0에 가까움 | NDWI 임계값 한계 | `수체(SCL 기준)` 값을 볼 것 |
| 엉뚱한 지역이 분석됨 | 지오코딩 오매칭 | 더 구체적으로(`Greenville, California`) |
| 서술 없이 수치만 나옴 | 품질 경고가 붙음 | 의도된 동작 — 데이터를 의심할 것 |

## 6. 원리적으로 안 되는 것 (미구현 아님)

- **건물 화재를 dNBR로 평가** — NBR은 식생 연소용입니다. 건물 화재는 `합성 변화`로 보세요.
  (인천 쿠팡물류센터 화재는 장마 구름으로 장면 자체가 없어 검증 불가였습니다)
- **주/국가 단위 광역 변화 스캔** — 현재 변화 탐지는 픽셀 단위·소규모(≤0.5°) 전용입니다.
  광역은 Clay 임베딩 기반 스캔이 필요하며 **미구현**입니다.
- **후속 질문의 맥락 유지** — 모든 호출이 단일턴이라 `거기 수체는?` 같은 질문은 맥락을 잃습니다.
  장소를 매번 명시하세요.
