## 1. 개요

경찰청 **도시교통정보센터(UTIC)** 개방데이터를 받아, 기존 GeoAerospace 지도 플랫폼에 **도심 교통 레이어**로 통합했다. 기존 CCTV·소통이 ITS(국토부) 고속도로·국도 중심이라 **도심이 비어 있던 빈틈**을, 경찰청 데이터(돌발·신호제어 교차로)로 메우는 것이 목표.

핵심 성과 두 가지:
1. **신호제어 교차로 레이어** — data.go.kr `CrossRoadInfoService`로 서울 398개 실데이터 표출
2. **돌발상황 레이어** — 경찰청 UTIC이 서버 IP 인증이라 배포가 까다로워, **ITS `eventInfo`로 전환**해 기존 키로 실동작(18건)

과정에서 "무엇이 실제로 되는가"를 **엔드포인트를 직접 찔러 검증**하며 진행했다 — 문서 스펙과 실 응답이 자주 달랐기 때문(좌표 스케일, 응답 구조, 인증 방식 모두).

> 설계원칙 유지: **결정론 데이터 = 숫자(사실), VLM = 픽셀, LLM = 조율.** 이번 작업은 전부 결정론 레이어(좌표·속성)라 VLM 개입 없음.

## 3. 데이터 소스 조사 — "무엇이 실제로 되는가"

UTIC 개방데이터 PDF(7종)를 받아 각 엔드포인트를 직접 검사한 결과, **문서와 실제가 여러 지점에서 달랐다.**

### 3.1 UTIC 7종 실체 (직접 검사)
- **소통정보**: `telMap.do?key=&x=&y=&lev=` — ⚠️**렌더된 지도만, 링크별 속도 숫자 없음**("교통량 데이터는 제공하지 않음" 명시). ITS `trafficInfo`와 달리 숫자 그라운딩 불가 → **드롭**.
- **돌발정보**: `imsOpenData.do` — XML(오류는 JSON), 좌표 `locationDataX/Y` WGS84. **key+서버IP 인증**(미등록 IP는 `유효한 KEY값이 아닙니다`).
- **CCTV**: `cctvOpenData.do`(가이드 HTML)→내부 `getCctvInfoById.do`. 스트림은 `openDataCctvStream.jsp`(JSP 플레이어, HLS 직접 아님). `MOVIE` 필드로 Y=스트림/U=유고/그외=정지JPEG 분기.
- 안전정보(xlsx TAAS 사고다발), 도로위험예보(기상융합 JSON), 신호개방, TCS공사(CSV, 좌표없음).

### 3.2 소통 오버레이가 비viable인 이유 (중요)
telMap을 iframe으로 띄우려 했으나 **UTIC은 요청 IP로 인증** → iframe은 **최종 사용자 브라우저 IP**로 요청하는데 등록된 건 서버 IP뿐. 서버 프록시로 HTML을 중계해도 telMap 내부 AJAX/타일이 다시 브라우저에서 UTIC을 호출→실패. **헤드리스 렌더 스냅샷만 가능(무거움) + 숫자 무기여 → 드롭 결정.**

### 3.3 data.go.kr 경로 발견
UTIC 웹(key+IP)의 대안으로 **공공데이터포털(apis.data.go.kr)** 에 경찰청/국토부 교통 API가 다수 존재함을 확인:

| 데이터셋 | 번호 | 유형 | 비고 |
|---|---|---|---|
| 경찰청_교차로기반정보서비스 | 15056721 | REST(serviceKey) | ✅ 신호 좌표(구현) |
| 경찰청_교차로계획정보서비스 | 15056569 | REST | 신호계획(TOD) — 추후 |
| 경찰청_교통돌발정보서비스 | 15088841 | **LINK+key/IP** | ⚠️ UTIC imsOpenData 재등록(IP 필요) |
| 경찰청_교통 CCTV 영상 정보 | 15148511 | — | 미탐색 |
| 경찰청_교통소통정보서비스 | 15087120 | — | 숫자 여부 미확인 |
| 국토교통부_돌발상황정보 | 15040465 | REST | ITS 계열 |

→ **경찰청 돌발은 data.go.kr(15088841)도 "LINK+IP"** 라 IP 회피 불가. 그래서 돌발은 ITS `eventInfo`로 방향 전환.

## 4. 구현 ① 돌발상황 레이어 (UTIC 시도 → ITS eventInfo 전환)

### 4.1 1차: UTIC imsOpenData (커밋 `1922741`)
- `lib/server/fetchIncidentUtic.ts`: imsOpenData XML/JSON 양파싱, `locationDataX/Y`(경/위도), `incidentTitle`·`incidenteTypeCd`·`controlType`·`important` 등. **제목 키워드로 유형 분류**(코드정의서 비의존 — 사고/공사/통제/행사/기상).
- 인증 실패(IP 미등록)를 명시적으로 구분해 UI에 "미등록" 표기.
- 결과: 실데이터는 서버 IP 등록 필요 → 배포 난도 높음.

### 4.2 2차: ITS eventInfo로 실동작 (커밋 `26c7961`)
- `lib/server/fetchIncidentIts.ts`: ITS `openapi.its.go.kr:9443/eventInfo` — **기존 ITS_API_KEY로 서버 IP 없이** 전국 돌발.
- 필드: `type`(도로종류)·`eventType`(작업/교통사고)·`coordX/Y`(WGS84 그대로)·`roadName`·`roadDrcType`·`lanesBlocked`·`message`·`startDate`.
- **폴백 로직**: 실키가 eventInfo엔 **401/일일한도(4001)** 걸림 → **데모키 `test`로 자동 폴백**(fetchCctv 철학). 실키=JSON/데모키=XML 양파싱.
- `eventType` → 종류 분류, `message`=제목, `roadDrcType + lanesBlocked`=통제정보.
- 팝업 출처를 소스에 따라 동적 표기, 힌트 `미등록`→`N 샘플`.

## 5. 구현 ② 신호제어 교차로 레이어 (data.go.kr)

### 5.1 엔드포인트·인증
- `apis.data.go.kr/1320000/CrossRoadInfoService/getCrossRoadInfoList`(https·기존 allowlist 재사용).
- ⚠️ **UTIC 웹키가 아니라 data.go.kr 서비스키**(`DATA_GO_KR_SIGNAL_KEY`). 프로젝트의 `KMA_SERVICE_KEY`(GK2A용)와 **동일 data.go.kr 계정** — 단, 서비스별 **활용신청** 필요.
- 신청 직후 `Forbidden`(전파 지연) → 수분 후 `NORMAL_SERVICE` 활성.

### 5.2 실 응답으로 드러난 함정 3종 (커밋 `8f584b0`)
문서 스펙과 실 API가 모두 달라, 실데이터로 검증하며 수정:

| 항목 | 문서/가정 | 실제 | 대응 |
|---|---|---|---|
| 응답 구조 | `{response.body.items.item}` | **`[{헤더},{행}…]` 평탄 배열** | 헤더 원소 필터링 |
| 좌표 스케일 | ÷1e6 | **÷1e7**(`1270769729`→127.07) | 경도·위도 **각자 기대범위 정규화** |
| 페이지 | numOfRows 자유 | **100 캡** | PER=100 페이지네이션 |

※ 위도(`375581376`)는 "<1000까지 나누기"로는 **375.58에서 오정지** → 위도 상한(≈40)으로 별도 정규화해야 37.55.

### 5.3 좌표·커버리지
- `REGION_CD·INT_NO·INT_NM·X_COORD·Y_COORD·UPD_DTIME`.
- REGION_CD **L01=서울**(문서의 인천·대구보다 확장). L27=대구, L29=인천 매핑.

## 6. 구현 ③ UI — 레이어레일·팝업

### 6.1 레이어레일 재정렬 + 접기/펼치기 (커밋 `c87680f`)
- **재정렬**: 성격별 그룹 소제목 — 기본(궤도·위성·항공·지형) / **관측**(산불) / **교통 · UTIC/ITS**(CCTV·돌발·신호).
- **접기/펼치기**: GeoAgent 런처 패턴 — 접으면 좌상단 `≡ 레이어 N`(활성 수 배지), 펼치면 헤더 접기 버튼.

### 6.2 드래그·접기 팝업 컴포넌트 (커밋 `248c1e9`·`bff94a8`)
- `components/hud/DraggablePopup.tsx`: CCTV 팝업의 **드래그**(헤더 잡고 이동) + GeoAgent식 **접기/펼치기**(`–`/`+`) + 닫기(`✕`) 통합.
- **앵커 기반 기본 위치**: `defaultPos`가 left/right/top/bottom 조합을 받아, **bottom/right 앵커면 창 크기가 바뀌어도 그 모서리에 고정**(드래그 전까지). 드래그 시 절대 left/top으로 전환.
- 기본 위치: 신호=좌하단 `{left:224, bottom:18}`, 돌발=`{left:212, top:74}`, CCTV=우하단 `{right:366, bottom:20}` — 서로·TrackCard·GeoAgent와 비겹침.

## 8. 함정·교훈 모음

- **문서 ≠ 실제**: 신호 좌표 스케일(문서 ÷1e6 / 실 ÷1e7), 응답 구조(nested vs 평탄배열), numOfRows 캡(100) — **엔드포인트를 직접 찔러 검증**이 필수.
- **인증 3종 구분**: UTIC=key+**서버IP**(브라우저/배포 난도↑), data.go.kr=serviceKey+**서비스별 활용신청**(신청 후 전파 지연 `Forbidden`), ITS=apiKey(+데모키 `test` 폴백). 같은 "경찰청 돌발"도 UTIC/data.go.kr 모두 IP-게이트.
- **IP 인증 iframe 불가**: 요청 주체가 브라우저 IP라 서버 등록으로 해결 안 됨(소통 오버레이 드롭 근거).
- **키 실패 폴백**: ITS 실키가 eventInfo엔 401/한도(4001)여도 데모키로 전국 데이터 확보 — 소스별로 "무엇이 실제 되는지"에 맞춰 우회.
- **좌표 정규화는 축별로**: 위경도 자릿수가 달라 단일 임계(<1000)면 위도가 10× 틀림. **경도 상한(≈133)·위도 상한(≈40) 개별 적용.**
- **React setState 배치**: 클릭 직후 같은 틱에서 DOM을 읽으면 리렌더 전이라 "안 바뀐 것처럼" 보임 → **검증은 다음 틱/별도 호출로**.
- **globe 검증 제약**: GPU 컴포짓으로 **스크린샷 타임아웃**, 맵 인스턴스가 페이지 JS로 안정 접근 안 됨 → 힌트 카운트·API 응답·콘솔로 대체 검증.
- **준수사항**: "경찰청 도시교통정보센터(UTIC) 제공" 출처 표기 의무, 효력 1년.

## 10. 미구현·다음 단계

- **신호계획(TOD) 팝업 조인**: `PlanCrossRoadInfoService/getPlanCRRSInfo`(15056569)를 INT_NO로 조인 → 교차로 클릭 시 현재 신호주기(신호대기≠정체 판독 근거).
- **교통안전정보(TAAS 사고다발)**: xlsx → PostGIS 공간검색(레인②) + RAG 그라운딩. `describe_region` 안전 강화. (실제 안전 xlsx 파일 필요 — 받은 CCTV xlsx는 빈 파일).
- **CCTV data.go.kr(15148511)**: serviceKey로 IP-게이트 JSP 없이 CCTV — 단 영상 재생 이질성(HLS/JPEG) 잔존.
- **ITS 실키 eventInfo 권한**: 현재 401/한도로 데모키 폴백 중 → 실키 eventInfo 활용신청 시 bbox 정확 데이터.
- **소통 숫자화 확인**: 경찰청 교통소통정보서비스(15087120)가 링크 속도 숫자를 주면 드롭했던 소통을 숫자로 부활(VLM 그라운딩).
