## 1. 위성 궤도 전파 (SGP4/SDP4)

### 논문 근거
- **Hoots, F. R. & Roehrich, R. L. (1980).** *Spacetrack Report No. 3: Models for Propagation of NORAD Element Sets.* — SGP4/SDP4 원 사양(공개 FORTRAN).
- **Vallado, D. A., Crawford, P., Hujsak, R. & Kelso, T. S. (2006).** *Revisiting Spacetrack Report #3.* **AIAA 2006-6753.** — 현대 표준 참조 구현(버그 수정). 현존 대부분의 라이브러리가 이 코드에서 파생.

### 채택 기술
- **satellite.js**(Vallado 참조 구현의 JS 포트) — 브라우저/Node 공용.
- TLE는 **SGP4 전용으로 피팅된 평균 궤도요소**이므로 반드시 대응 모델로만 전파(일반 케플러 계산 금지).

### 구현 내용 — `lib/orbit.ts`
```ts
const satrec = satellite.twoline2satrec(tle1, tle2);
const periodMin = (2 * Math.PI) / satrec.no;   // 평균운동(rad/min) → 주기
const pv = satellite.propagate(satrec, date);  // → ECI(TEME) 위치·속도
```
- **서버·클라이언트 이중 실행**: 서버는 정적 산출물(궤도 링), 클라이언트는 매 프레임 현재 위치 전파.
- 한 주기를 `steps=180`(3D는 256)으로 샘플링해 궤도 링·지상궤적 동시 생성.

### 실습 · 검증 — `lib/orbit.test.ts` (`npm test`)
**Vallado sgp4-ver 표준 검증 세트(catalog 00005, t=0, TEME)** 와 직접 대조:
```
TLE: 1 00005U 58002B   00179.78495062 ...
공표 r = (7022.46529266, -1400.08296755, 0.03995155) km
공표 v = (1.893841015, 6.405893759, 4.534807250) km/s
```
- 결과: **위치 오차 < 0.01 km(10 m), 속도 오차 < 0.001 km/s** 이내 일치 → satellite.js가 참조 구현과 등가임을 입증.
- ISS 불변식(주기 85–100분, 경사각 51.0–52.0°, 고도 150–600 km, 지상궤적 자오선 분할) 통과.
- **테스트 2/2 통과.**

## 2. 좌표계 변환 (ECI → ECEF → 측지)

### 논문 근거
- **Vallado, D. A.** *Fundamentals of Astrodynamics and Applications* — 프레임 변환 표준 텍스트.
- GMST(그리니치 평균 항성시) 회전, WGS-84 타원체 역변환.

### 구현 내용
```ts
const gmst = satellite.gstime(date);              // 지구 자전각
const ecf  = satellite.eciToEcf(posEci, gmst);    // ECI → ECEF
const geo  = satellite.eciToGeodetic(posEci, gmst); // → lat/lon/alt(WGS-84)
```
- 궤도 링은 **고도 유지**, 지상궤적은 **height=0 투영** + **자오선(±180°) 세그먼트 분할**(래핑 아티팩트 방지).

### 실습 · 검증
- ISS 실측 대조: **경사각 51.6°**(공식값 일치), **주기 92.9~93.0분**(일치), **고도 427km**(실제 ISS 고도대 일치), 속도 7.66 km/s.
- 정확도 한계 인지: 현재 GMST 근사(GAST·세차·장동·극운동 미적용) → 수십~수백 m 오차. 고도화 A2에서 IERS 정밀 프레임으로 개선 예정.

## 4. 실축척 3D 궤도 뷰 (관성계)

### 논문 근거 / 설계 근거
- 위 CZML 논문의 **관성계 시간-동적 표현** 개념. CesiumJS의 `ReferenceFrame.INERTIAL` 방식과 동일한 원리를 Three.js로 구현.

### 채택 기술 · 핵심 아이디어
- **satellite.js가 반환하는 ECI 좌표를 그대로 3D 씬에 배치**(측지 변환 없이) → 궤도는 관성계에 고정, **지구만 GMST로 자전**. 이것이 "진짜 관성계 실축척 궤도".
- 스케일: **1 scene unit = 1000 km**(지구 반경 6.371) → LEO 궤도(6.7~7.2)가 지구에 밀착한 **실제 축척**.

### 구현 내용 — `components/SpaceView.tsx`
```ts
const eciToScene = p => new THREE.Vector3(p.x/1000, p.z/1000, -p.y/1000); // Z-up→Y-up
earth.rotation.y = satellite.gstime(now);   // 지구만 자전
```
- 대기 **프레넬 림글로우 셰이더**(BackSide + Additive), 위경도 wireframe, 별필드 2,200점, 센서 콘, **OrbitControls** 자유 시점, Raycaster 위성 클릭.

### 실습 · 검증
- 3D 전환 시 **콘솔 에러 0**, WebGL 캔버스 956×910 활성, MapCanvas 언마운트 확인(GPU 정리).

## 5. globe 위 3D 위성 모델 (MapLibre custom layer)

### 규격 근거
- **MapLibre GL JS v5 custom layer** 공식 규격: `type:'custom'`, `renderingMode:'3d'`, `map.transform.getMatrixForModel(lngLat, alt)`, `args.defaultProjectionData.mainMatrix` (globe 투영 공식 지원, 첫 파티 예제 존재).

### 구현 내용 — `lib/three/orbitalLayer.ts`
```ts
camera.projectionMatrix.copy(mainMatrix).multiply(getMatrixForModel([lng,lat], alt));
renderer.render(scene, camera);   // 객체별 렌더(재사용 단일 scene)
```
- 절차적 위성 모델(본체 + 태양전지판 2 + 붐 + 안테나), 추적 대상 **센서 콘**(정점=위성, -Z nadir 개방).
- 궤도 링은 deck.gl 유지(globe 전역 튜브는 단일 행렬로 불가 → Three.js는 위치 국소 객체 담당).

### 실습 · 검증
- `node_modules/maplibre-gl` dist에서 **`getMatrixForModel` 존재 확인**(v5.24.0) → API 가용성 입증. 런타임 에러 0.

## 8. LLM 자연어 지도 제어 (에이전트)

### 논문 근거
- **Yao, S. et al. (2022).** *ReAct: Synergizing Reasoning and Acting in Language Models.* **arXiv:2210.03629** — 추론↔도구호출↔관찰 루프.
- Schick et al., *Toolformer* (arXiv:2302.04761) — 도구 사용 학습.
- **Roberts, J. et al. (2023).** *GPT4GEO* (arXiv:2306.00020) — **LLM의 좌표 환각 증거** → 본 구현의 지오코딩 강제 근거.
- Manvi et al., *GeoLLM*, **ICLR 2024** (arXiv:2310.06213) — LLM의 지리 prior.

### 채택 기술
- 백본: **Qwen3-8B (Apache-2.0, Ollama 로컬 GPU)** — Ollama native `/api/chat`(tools, **`think:true`**).
- **결정론적 의도 해석 그라운딩 레이어**(설계서 §4.5 그라운딩 원칙) — 소형 모델의 도구선택 변동성 보정.
- **지오코딩 강제**: 모델은 지명 문자열만 전달, 좌표는 시스템이 변환(도시 테이블 + Nominatim) → GPT4GEO가 지적한 좌표 환각 차단.

### 구현 내용
- `lib/agentTools.ts` — 도구 스키마(`fly_to_place`·`select_satellite`·`toggle_layer`) + 시스템 프롬프트.
- `app/api/agent/route.ts` — Ollama 도구호출 프록시.
- `lib/agent.ts` — `resolveIntent`(위성명/레이어+동사/도시/이동동사 확정 추출) → `execTool` → 지도 조작(`mapBus`).
- `components/hud/ChatDrawer.tsx` — 계획 스텝(✓ 실행 도구)·결과 표시.

### 실습 · 검증 (중요 발견 포함)
- **Qwen3-8B의 도구선택이 매우 불안정함을 실측**:
  - "서울로 이동" → `toggle_layer(terrain)` (오분류)
  - "도쿄 상공" → `fly_to_place`이지만 **서울 좌표**를 복사(프롬프트 예시 오염)
  - 상태 라인("추적=ISS")에 편향되어 모든 요청을 `select_satellite(ISS)`로 호출
  - `think:false`가 `think:true`보다 **악화**(Qwen3는 thinking 기반 도구호출로 학습됨)
- → **결론:** 프롬프트 엔지니어링만으로는 부족. **결정론적 그라운딩 레이어 필수**(설계서 §4.5가 예견한 8B 한계). 프로덕션은 Qwen3-30B-A3B 승급 권고.
- **최종 E2E 성공**: 챗 "도쿄 상공을 보여줘" → `✓ fly_to_place(도쿄)` → 지도 이동 + "완료했습니다".

## 10. 태양 위치 · 식(eclipse) — 고도화 D2

### 근거
- **저정밀 태양 위치 표준식**(Astronomical Almanac 계열, 정확도 ~0.01°): 평균황경 L, 평균근점이각 g, 황도경도 λ, 황도경사 ε.
- **원통 umbra 근사** — 위성이 태양 반대쪽이고 태양-지구 축으로부터 수직거리 < 지구 반경이면 식.

### 구현 — `lib/orbit.ts`
```ts
export function sunEci(date) { /* L,g,λ,ε → 단위벡터 */ }
export function isEclipsed(pos, date) {
  const s = sunEci(date); const d = pos·s;
  if (d >= 0) return false;                 // 태양 쪽
  return |pos - d·s| < 6371;                // 그림자 원통 내부
}
```
- 3D 뷰: 태양 방향으로 DirectionalLight 배치 → **실시간 주야 터미네이터**. 식이면 위성 색 ×0.28.

### 실습 · 검증
- TrackCard에 **"식(그림자)"** 표시 확인(ISS가 실제 그림자 구간). 일조 시 앰버 "일조".

## 11. TLE 신선도 · 오차 정량화 — 고도화 D1

### 근거
- TLE는 **하루 약 1~3 km** 오차 누적(제안서 §4.1) → "오차 0"은 불가, **최소화 + 정량화**가 정석.

### 구현
```ts
tleEpochMs = (satrec.jdsatepoch + jdsatepochF - 2440587.5) * 86400000;
ageDays = (now - epoch)/86400000;  estErrKm = max(1, ageDays * 2);
```
- TrackCard 배지: **`TLE 0.6d · ±1km`**, 색상 코딩(<2d 녹 / 2~7d 황 / >7d 적). 서버 캐시 TTL 4h→**2h** 단축.

### 실습 · 검증
- 실제 SatNOGS TLE(epoch 2026-07-15)로 **0.6d · ±1km** 산출 → 신선도 정량화 동작 확인.

## 12. 시간 제어 (가상 시계) — 고도화 D3

### 구현 — `lib/simClock.ts`
- anchor 방식 가상 시계: `now = anchorSim + (real - anchorReal) × rate`, 일시정지·배속(1/60/3600×)·seek(±1h)·reset.

## 13. 회복탄력성 · 보안 (구현분석 차용)

### 근거
- 내부 구현분석 `0716_OSIRIS_구현기술_분석.md`의 검증된 관용구를 선별 채택(제안서 §4.8).

### 구현
- `lib/server/safeFetch.ts` — **호스트 allowlist + https 강제 + 타임아웃 + `redirect:'error'`**(public→private 리다이렉트 우회 차단) + User-Agent.

- **명시적 거부**: OSIRIS의 `stealthFetch`(위조 `X-Forwarded-For`로 레이트리밋 우회)는 **API ToS 위반 소지**로 채택 금지. 정당한 캐시·백오프·다중 소스로 한도 관리.

### 실습 · 검증
- **CelesTrak이 이 네트워크에서 완전 차단**(`http=000`, 15초 타임아웃) → **SatNOGS 폴백 자동 동작**(`http=200`, 1.7초)으로 실데이터 확보. 설계한 회복탄력이 실제 장애에서 작동함을 입증.
